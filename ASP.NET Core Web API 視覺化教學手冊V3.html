ASP.NET Core Web API 實戰教學

專案實作：Employees 與 Departments 管理系統

1. 請求生命週期：深度管線 (Request Pipeline)

一個 HTTP 請求從進入到回傳，在 .NET 內部會經歷以下精密的環節：

01 客戶端 (Client)
02 網頁伺服器 (Kestrel)
03 Middleware (Exception/Auth)
04 端點路由 (Endpoint Routing)
05 模型綁定 (Model Binding)
06 模型驗證 (Model Validation)
07 過濾器管線 (Filter Pipeline)
08 控制器 Action (Controller)
09 服務層 (Service)
10 倉儲層 (Repository)
11 DbContext (EF Core)
12 資料庫 (Database)
第 13 步：Response 回傳時，會依原路「反向」通過 Middleware 進行最終處理（如日誌紀錄）。

4. Model 配置與自動驗證 (Validation)

Employee.cs 配置

using System.ComponentModel.DataAnnotations;

public class Employee
{
    [Key]
    public int EmployeeId { get; set; }

    [Required] // 觸發 Model Validation
    [StringLength(100)]
    public string EmployeeName { get; set; } = "";

    [Required]
    public string Department { get; set; } = "";
    
    public DateTime DateOfJoining { get; set; }
    public string PhotoFileName { get; set; } = "";
}

Model 在 Pipeline 的關鍵期

當請求到達第 5 步時，.NET 會執行：

  • Model Binding：將 JSON 自動轉為 Employee 物件。
  • Model Validation：檢查 [Required] 等標記。若驗證失敗，管線會在此阻斷並回傳 400 錯誤。

5. 架構核心：Interface 與 Repository

IEmployeeRepository.cs (契約)

public interface IEmployeeRepository
{
    IEnumerable<Employee> GetEmployees();
    Employee AddEmployee(Employee emp);
}

EmployeeRepository.cs (實作)

public class EmployeeRepository : IEmployeeRepository
{
    // 注入 DbContext 直接對話資料庫 (第 11 步)
    private readonly EmployeeDbContext _context;
    public EmployeeRepository(EmployeeDbContext ctx) => _context = ctx;

    public Employee AddEmployee(Employee emp)
    {
        _context.Employees.Add(emp);
        _context.SaveChanges();
        return emp;
    }
}

6. 實戰演練：從請求到回復的完整路由

以「新增員工」為例，觀察每一層級的轉變：

Step 1-3：Client 到 Middleware

請求通過 Kestrel，Exception Middleware 會在此捕捉整個流程的錯誤。

Step 4-6：路由、綁定與驗證

系統判定 URL 與 POST 動詞，將 JSON 內容「綁定」到 Model，並執行屬性檢查。

Step 7-8：Filter 到 Controller Action

通過 Filter（如日誌記錄）後，進入 Controller。Controller 呼叫 Service 介面。

Step 9-11：Service、Repository 到 DbContext

Service 處理邏輯，Repository 呼叫 DbContext，EF Core 生成 SQL 並準備發送。

Step 12-13：資料庫存取與 Response 反向傳回

SQL 執行完成後，結果封裝成 Response，沿路反向穿過所有 Middleware 回到 Client。

Response Status: 201 Created
Location: /api/employees/1
