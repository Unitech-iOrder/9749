ASP.NET Core Web API 實戰教學

專案實作：Employees 與 Departments 管理系統

1. 請求的生命週期 (Life of a Request)

當客戶端發送一個請求時，路由系統（Routing）是決定請求去向的導航儀：

客戶端
→
Middleware
過濾與授權
→
Routing
匹配控制器
→
Controller
解析請求參數
→
Service
執行 CRUD
→
資料庫

2. 開發工具與快速啟動 (VS Code CLI)

使用以下指令快速建立您的 Employee 管理系統：

# 1. 建立 Web API 專案
dotnet new webapi -n EmployeeManager

# 2. 安裝必要的 SQL Server 套件 (如果使用 EF Core)
dotnet add package Microsoft.EntityFrameworkCore.SqlServer

# 3. 啟動熱更啟動服務
dotnet watch run

3. CRUD 流程詳解：路由匹配到資料庫存取

我們以「新增一位員工 (POST)」為例，說明資料如何流過每一個環節：

步驟 A：客戶端發出請求

發送目標 URL 為 /api/Employees，動詞為 POST。

{
  "EmployeeName": "李大明",
  "Department": "IT 部門",
  "DateOfJoining": "2026-02-10",
  "PhotoFileName": "daming.png"
}

步驟 B：路由引擎匹配 (Routing Match)

路由引擎會檢查 Controller 上的屬性標記：

  • [Route("api/[controller]")]：將 URL 段 Employees 映射到 EmployeesController。
  • [HttpPost]：確認請求動詞為 POST，找到正確的處理方法。
// 路由決策過程
URL: /api/Employees + Verb: POST => 執行 EmployeesController.Post()

步驟 C：Controller 接收並呼叫 Service

API 接收 JSON 並自動轉換為 Employee 物件，隨即交給 Service 層處理業務邏輯。

[HttpPost]
public IActionResult Post(Employee emp) 
{
    // 調用服務層
    var result = _service.AddEmployee(emp);
    return CreatedAtAction(nameof(Get), new { id = result.EmployeeId }, result);
}

步驟 D：Service 執行業務驗證

這裏會檢查 Departments 資料表，確認部門名稱有效，這就是所謂的商業規則驗證。

public Employee AddEmployee(Employee emp) 
{
    // 跨表檢查：確認部門是否存在於 Departments 表
    var deptExists = _deptService.GetByName(emp.Department);
    if (deptExists == null) throw new Exception("部門不存在");

    return _repository.Insert(emp); // 呼叫資料存取層
}

步驟 E：資料庫執行 CURD

SQL Server 接收指令，完成 Employees 表的新增並回傳 ID。

INSERT INTO [Employees] (EmployeeName, Department, DateOfJoining, PhotoFileName)
VALUES ('李大明', 'IT 部門', '2026-02-10', 'daming.png');
-- DB 回復成功，回傳 Identity ID

步驟 F：回復客戶端

API 回傳 HTTP 201 狀態碼與 JSON 結果，完成整個請求流程。

4. 如何測試 (Testing)

使用 Swagger UI

執行專案後存取 /swagger。您可以視覺化地填寫欄位並觀察回傳狀態碼 (200, 201, 404)。

使用 VS Code REST Client

在 .http 檔案中撰寫：
GET http://localhost:5000/api/Employees
