ASP.NET Core Web API 實戰教學

專案實作：Employees 與 Departments 管理系統

1. 請求的生命週期 (Life of a Request)

當客戶端發送一個請求（例如新增員工）時，它會依序經過以下管線：

客戶端
→
Middleware
驗證、CORS
→
Controller
路由分發
→
Service
業務邏輯
→
資料庫

2. 開發工具與快速啟動 (VS Code CLI)

使用以下指令快速建立您的 Employee 管理系統：

# 1. 建立 Web API 專案
dotnet new webapi -n EmployeeManager

# 2. 安裝必要的 SQL Server 套件 (如果使用 EF Core)
dotnet add package Microsoft.EntityFrameworkCore.SqlServer

# 3. 啟動熱更啟動服務
dotnet watch run

3. CRUD 流程詳解：從請求到回復

我們以「新增一位員工 (POST)」為例，說明資料如何在這兩個表格間流動：

步驟 A：客戶端發出請求 (HTTP POST)

前端發送包含員工資訊的 JSON 資料。注意 Department 必須匹配現有的部門名稱。

{
  "EmployeeName": "李大明",
  "Department": "IT 部門",
  "DateOfJoining": "2026-02-10",
  "PhotoFileName": "daming.png"
}

步驟 B：Controller 接收並驗證

API 透過 EmployeesController 接收 DTO，並呼叫 Service 層。

[HttpPost]
public IActionResult Post(Employee emp) 
{
    var result = _service.AddEmployee(emp);
    return CreatedAtAction(nameof(Get), new { id = result.EmployeeId }, result);
}

步驟 C：Service 處理業務邏輯

在 EmployeeService 中，我們會先確認 Departments 表格中是否存在該部門，確保資料完整性。

public Employee AddEmployee(Employee emp) 
{
    // 檢查部門是否存在
    var deptExists = _deptService.GetAll().Any(d => d.DepartmentName == emp.Department);
    if (!deptExists) throw new Exception("無效的部門名稱");

    return _repository.Save(emp); // 進入資料庫層
}

步驟 D：資料庫回復 (CRUD Result)

SQL Server 執行 INSERT 指令，並回傳自動生成的 EmployeeId。

INSERT INTO [Employees] (EmployeeName, Department, DateOfJoining, PhotoFileName)
VALUES ('李大明', 'IT 部門', '2026-02-10', 'daming.png');
-- 回傳：EmployeeId = 1

步驟 E：客戶端收到 201 Created 回復

最終 API 將帶有 ID 的完整物件回傳，前端即可更新 UI 顯示新員工。

4. 如何測試 (Testing)

使用 Swagger UI

執行專案後存取 /swagger。您可以視覺化地填寫欄位並觀察回傳狀態碼 (200, 201, 404)。

使用 VS Code REST Client

在 .http 檔案中撰寫：
GET http://localhost:5000/api/Employees
