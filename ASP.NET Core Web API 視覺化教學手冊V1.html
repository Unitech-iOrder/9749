→
Middleware

→

2. 資料表欄位詳細配置 (Table Schema)

Departments (部門表)

欄位名稱 資料類型 條件限制 說明
DepartmentId int PK, Identity(1,1) 部門唯一識別碼
DepartmentName nvarchar(100) NOT NULL 部門名稱

Employees (員工表)

欄位名稱 資料類型 條件限制 說明
EmployeeId int PK, Identity(1,1) 員工唯一識別碼
EmployeeName nvarchar(100) NOT NULL 員工全名
Department nvarchar(100) NOT NULL 所屬部門名稱
DateOfJoining datetime2(7) NOT NULL 入職日期

3. 開發工具與快速啟動 (VS Code CLI)

在開始寫程式之前，必須先透過 .NET CLI 建立專案骨架：

# 1. 建立 Web API 專案 (這是開發的第一步)
dotnet new webapi -n EmployeeManager

# 2. 進入專案目錄
cd EmployeeManager

# 3. 安裝必要的 SQL Server 套件
dotnet add package Microsoft.EntityFrameworkCore.SqlServer

# 4. 啟動熱更啟動服務
dotnet watch run

4. Model 實體模型配置 (C# Classes)

專案建立完成後，我們在 Models/ 資料夾下定義與資料表對應的 POCO 類別：

Employee.cs 配置

using System.ComponentModel.DataAnnotations;

public class Employee
{
    [Key]
    public int EmployeeId { get; set; }

    [Required]
    [StringLength(100)]
    public string EmployeeName { get; set; } = "";

    [Required]
    public string Department { get; set; } = "";

    public DateTime DateOfJoining { get; set; }

    public string PhotoFileName { get; set; } = "";
}

關鍵註解說明

  • [Key] 標記主鍵，Entity Framework 會自動識別並關聯至 DB 的 PK。
  • [Required] 模型驗證器會確保傳入的資料不為 Null。
  • [StringLength] 限制字串長度，與 DB 的 nvarchar(100) 達成同步。

5. CRUD 流程詳解：路由匹配到資料庫存取

我們以「新增一位員工 (POST)」為例，說明資料如何流過每一個環節：

步驟 A：客戶端發出請求

發送目標 URL 為 /api/Employees，動詞為 POST。

{
  "EmployeeName": "李大明",
  "Department": "IT 部門",
  "DateOfJoining": "2026-02-10",
  "PhotoFileName": "daming.png"
}

步驟 B：路由引擎匹配 (Routing Match)

路由引擎會檢查 Controller 上的屬性標記，精準定位處理邏輯。

URL: /api/Employees + Verb: POST => 執行 EmployeesController.Post()

步驟 C：Controller 接收並呼叫 Service

API 接收 JSON 並根據 Model 配置 自動驗證與轉換為 C# 物件。

[HttpPost]
public IActionResult Post(Employee emp) 
{
    // 此處的 emp 實例就是 Model 
    var result = _service.AddEmployee(emp);
    return CreatedAtAction(nameof(Get), new { id = result.EmployeeId }, result);
}

步驟 D：Service 執行業務驗證 (操作 Model)

Service 拿到 Model 物件後，進行最後的邏輯檢查與資料庫保存。

public Employee AddEmployee(Employee emp) 
{
    // 使用 Model 中的屬性進行檢查
    var deptExists = _deptService.GetByName(emp.Department);
    if (deptExists == null) throw new Exception("部門不存在");

    return _repository.Insert(emp); // 將 Model 傳入持久層
}

步驟 E：資料庫執行 CURD (映射 Model)

INSERT INTO [Employees] (EmployeeName, Department, DateOfJoining, PhotoFileName)
VALUES ('李大明', 'IT 部門', '2026-02-10', 'daming.png');

步驟 F：回復客戶端

API 回傳 HTTP 201 狀態碼與 JSON 結果，完成整個請求流程。
