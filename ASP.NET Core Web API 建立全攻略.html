ASP.NET Core Web API 建立全攻略

專案實作：EmployeeDB 員工管理系統

從零開始：六大建立步驟

1

初始化專案

打開終端機，執行以下 CLI 指令建立 Web API 模板專案。

# 建立專案
dotnet new webapi -n EmployeeApi

# 進入資料夾並用 VS Code 打開
cd EmployeeApi
code .
2

建立資料模型 (Models)

建立 Models 資料夾，並根據資料庫欄位定義實體類別。

Models/Employee.cs 
public class Employee {
    public int EmployeeId { get; set; }
    public string EmployeeName { get; set; } = "";
    public string Department { get; set; } = "";
    public DateTime DateOfJoining { get; set; }
    public string PhotoFileName { get; set; } = "";
}
3

定義服務層 (Service Layer)

建立 Interfaces 與 Services 資料夾。這是處理商務邏輯的地方。

Interfaces/IEmployeeService.cs 
public interface IEmployeeService {
    IEnumerable<Employee> GetAll();
    void Add(Employee emp);
}
4

註冊服務 (Dependency Injection)

在 Program.cs 告訴 .NET 如何注入這些服務。

// Program.cs 內容
var builder = WebApplication.CreateBuilder(args);

// 註冊 Service (生命週期為 Scoped)
builder.Services.AddScoped<IEmployeeService, EmployeeService>();

builder.Services.AddControllers();
5

建立控制器 (Controller)

在 Controllers 資料夾下建立 API 的進入點。

Controllers/EmployeeController.cs 
[Route("api/[controller]")]
[ApiController]
public class EmployeeController : ControllerBase {
    private readonly IEmployeeService _service;
    public EmployeeController(IEmployeeService service) => _service = service;

    [HttpGet]
    public IActionResult Get() => Ok(_service.GetAll());
}
6

執行與測試 (Testing)

這是完成開發後最重要的一步。我們有兩種主要測試方式：

方式 A：Swagger UI

  • 1. 執行 dotnet run
  • 2. 瀏覽器開啟 /swagger 頁面
  • 3. 點擊 Try it out 並執行

方式 B：VS Code REST Client

  • 1. 建立 test.http 檔案
  • 2. 輸入以下請求格式：
GET http://localhost:5000/api/employee

如何測試 POST (新增員工)

測試新增功能時，JSON 的日期格式非常重要。請參考以下範例：

# 請求內容 (JSON Body)
{
  "EmployeeName": "王小明",
  "Department": "IT 部門",
  "DateOfJoining": "2026-02-10",
  "PhotoFileName": "wang.png"
}
